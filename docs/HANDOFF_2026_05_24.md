## 5. 확정된 정책

### 5.1 거래 상태값
- `OPEN` = 판매 중
- `RESERVED` = 예약 중
- `COMPLETED` = 거래 완료
- `DELETED` = 삭제됨 (soft delete, hard delete 금지)

CLOSED / CANCELED는 도메인에서 제거됨.

### 5.2 호가 표시
- 호가/거래 탭에 OPEN + RESERVED 표시
- COMPLETED + DELETED 제외
- RESERVED는 숨김이 아닌 진행 중 상태

### 5.3 상태 변경
- 판매자만 가능 (backend `sellerId == userId` guard 적용 완료)
- 구매자는 chip 표시만, 클릭 불가
- 같은 상태로 재변경 시 no-op (DB update X, SYSTEM 메시지 X)
- 점점점 메뉴는 상태 변경용 아님 — 신고/차단/나가기/알림용으로 재정의 예정

### 5.4 카운트 표시 정책

| 화면 | 조회 | 채팅 | 관심 |
|---|:-:|:-:|:-:|
| 판매글 상세 | O | O | O |
| 내 판매 항목 (filterSellerId != null) | O | O | O |
| 일반 거래 리스트 | X | O | O |
| 호가 하단 리스트 | X | O | O |

조회수는 판매자 관리 지표. 구매자 리스트에는 상태/채팅/관심이 더 중요.

### 5.5 채팅방 SYSTEM 메시지 톤
- 만남/택배/선입금/안전결제 등 특정 거래 방식 권장 금지
- "사용자 간 협의" / "주의가 필요" / "의심 시 신고·차단" 톤
- 금지 표현: "사기다", "응하지 마세요", "직거래하세요", "직접 만나서"

### 8.2 DB 설계 (검토용)

```sql
CREATE TABLE reports (
    report_id        VARCHAR(36) PRIMARY KEY,
    reporter_user_id VARCHAR(36) NOT NULL,
    target_type      VARCHAR(20) NOT NULL,  -- TRADE / CHAT_ROOM / USER
    target_id        VARCHAR(36) NOT NULL,
    reason           VARCHAR(30) NOT NULL,  -- SCAM / ABUSE / SPAM / INAPPROPRIATE / OTHER
    detail           TEXT,
    status           VARCHAR(20) NOT NULL DEFAULT 'PENDING',
    created_at       TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at       TIMESTAMP
);
CREATE INDEX idx_reports_target ON reports (target_type, target_id);
CREATE INDEX idx_reports_reporter ON reports (reporter_user_id);
```
