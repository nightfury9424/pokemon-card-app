## 1. 앱 정체성

**"포켓몬 카드를 주식처럼"** — 한국 포켓몬 카드 수집가/거래자를 위한 포트폴리오+시세+거래 통합 플랫폼.

현재 수집가 워크플로우: 번개장터 → 네이버 → eBay → scrydex → 엑셀 → 카카오 채팅을 **앱 하나로** 대체.

핵심 무기: **KO 추정가** — 글로벌 가격(EN/JP scrydex)에 환율+국내 계수를 적용해 한국 원화 시세를 실시간 추산. 경쟁 앱 어디에도 없는 기능.

### 타겟 유저
- 1차: 고레어 카드 보유/거래하는 진지한 수집가 (SSR~CHR)
- 2차: 시세 정보로 구매 타이밍을 잡는 스펙큘레이터
- 3차: 번개장터/네이버 통해 카드 사고파는 개인 거래자

## 카드 상세 설계 노트

2026-05-11 야간 리서치 결과, 카드 상세 리디자인은 `NestedScrollView + SliverAppBar + TabBarView` 구조로 진행한다. 탭은 `시세 | 내 자산` 2개를 기본으로 유지하고, 비소유자는 `시세`, 소유자는 `내 자산`을 기본 탭으로 연다.

핵심 이유:

- 현재 `card_detail_screen.dart`는 카드 히어로, 자산 등급, 시세 차트, 호가, 판매 목록이 단일 스크롤에 섞여 있어 소유자 손익과 비소유자 시세 탐색이 모두 흐려진다.
- 이미 상세 화면 전용 `GET /api/prices/cards/{cardId}/price-summary`가 KO/EN/JP 차트와 KO mid/low/high를 제공하므로 `시세` 탭 구현 리스크가 낮다.
- `AssetDto`는 `quantity`, `purchasePrice`, `cardStatus`, `gradingCompany`, `gradeValue`, `certNumber`, 앱 분석 점수, `isSelling`, `activeTradeId`를 제공하므로 `내 자산` 탭에서 보유 수량, 구매가, 현재가, 손익, 등급, 판매 상태를 구성할 수 있다.
- 카드 이미지는 TCGplayer식 hero로 크게 보여주되, 실제 카드 이미지는 화면 폭의 약 62%로 제한하고 100:140 비율을 유지한다.

주의할 점:

- `GET /api/trades/cards/summary`는 존재하지만 카드별 상세 요약용 `cardId` 파라미터가 없으므로, 상세 화면의 판매 목록은 현재처럼 `GET /api/trades?cardId={cardId}`를 사용한다.
- 환율과 적용 계수는 현재 `price-summary` DTO에 직접 노출되지 않는다. "환율 + 계수 표시"는 백엔드 응답에 `fxRate`, `coefficient`, `sourceMarket`, `sourceRawPrice`를 추가해야 정확히 구현할 수 있다.
- `quantity > 1`일 때 `purchasePrice`가 장당 가격인지 총액인지 데이터 의미가 애매하다. 내일 구현에서는 기존 포트폴리오 계산 흐름에 맞춰 장당 구매가로 표시하고, 장기적으로 lot 모델을 검토한다.

상세 설계 문서: [docs/CARD_DETAIL_REDESIGN.md](docs/CARD_DETAIL_REDESIGN.md)
