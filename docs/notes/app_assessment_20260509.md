## 1. 강점

**시세 파이프라인 다각화**
scrydex EN/JP 실시간 연동, 번개장터 크롤링, eBay API 교차검증을 조합한 3중 소스 구조. 단일 소스 의존 대비 오염 감지와 신뢰도 확보 측면에서 명확한 우위.

**가격 오염 방지 시스템 (금일 구현)**
PSA10 급락 감지 → eBay 교차검증 → 저장 스킵 흐름이 파이프라인에 붙었다. 판초 피카츄 사례처럼 단발성 사고를 반복 방지하는 방어 레이어가 생긴 것은 실질적 개선.

**어드민 도구 완성도**
대시보드, 카드 관리, 유저, 거래, 시세, 스캐너, 가격 이상 알림(`/alerts`)까지 운영에 필요한 도구가 갖춰져 있음. 외부 서비스 없이 자체 운영 가능한 수준.

**스캐너 기술 완성도**
DINOv2 + FAISS 기반 7,192 벡터 인식 파이프라인이 실동작함. 카드 인식 기술 자체는 수준급.

**KO 전용 예상가 체계**
EN scrydex × 환율 × 한국 시장 계수라는 독자적 KO 가격 산출 공식이 있고, 계수는 매일 자동 갱신됨. KO 가격 데이터를 자체적으로 만들어내는 앱은 국내에 거의 없음.

## 2. 약점 / 리스크

**437개 미매핑 카드**
scrydex EN/JP 참조가 없는 카드는 KO 예상가 산출 불가. 전체 카드 DB(9,728장) 대비 437개가 아직 가격 없는 상태. 고가 희귀 카드 중 미매핑이 포함되어 있을 가능성이 높음.

**eBay API 미검증**
`price_ebay.py`의 교차검증 로직이 코드로만 존재하고 실제 eBay Finding API 호출이 검증된 적 없음. 오염 감지 시 교차검증이 실패하면 방어 로직 전체가 무력화됨.

**번들 매물 필터 없음**
번개장터/NAVER_CAFE 수집 데이터에 묶음 판매(여러 카드 세트) 가격이 단품 가격으로 잘못 분류될 수 있음. 필터링 로직 미구현 상태.

**단일 개발자 의존 / 테스트 부재**
전체 코드베이스를 1인이 유지. 자동화 테스트 커버리지가 사실상 없음. 변경 시 regression을 수동으로 확인해야 해서 실수 발생 시 감지가 늦음. (오늘만 해도 DISTINCT ON 버그, popularityMultiplier 불일치, JP slug 오염 등 데이터 정합성 이슈가 여럿 발견됨.)

**scrydex 의존도**
KO 예상가의 핵심 소스가 scrydex EN/JP임. scrydex가 사이트 구조를 바꾸거나 접근을 차단하면 파이프라인 전체가 흔들림.

**crawl 잔여 데이터 4,697개 미처리**
번개장터 크롤 결과 중 라벨링/DB 적재가 안 된 건이 4,697개 남아 있음. 이 데이터가 쌓이기만 하고 실제 가격 DB에 반영되지 않으면 파이프라인 의미가 희석됨.

## 3. 데이터 품질

**KO 예상가 신뢰도: 중간**
- scrydex EN/JP 실시간 데이터 기반이라 원소스가 맞으면 KO도 합리적.
- 그러나 오늘 발견된 것처럼 scrydex가 잘못된 카드 데이터를 반환할 경우 KO까지 오염됨.
- guard 로직 추가로 이전보다 나아졌으나, eBay API 검증이 실운영 테스트 전이므로 완전히 신뢰하긴 이름.

**번개장터 실거래가 신뢰도: 낮음~중간**
- 번들 매물 필터 미구현. 단품 가격과 묶음 가격이 섞일 수 있음.
- 4,697개 미처리 잔량이 있어 최신성도 불확실.

**scrydex 오염 사례**
- 판초 피카츄 PSA10: $22,000 → $695 (다른 카드 데이터 덮어쓰기)
- 비크티니/라티아스 ex/뮤 ex RR: EN 컬럼에 JP slug 오염
- 메가리자몽Y ex: stale JP 가격으로 KO 과소산출

세 가지 유형의 오염이 이번 세션에서 한 번에 발견됨. 이미 알려진 케이스 외에도 조용히 오염된 카드가 더 있을 가능성을 배제할 수 없음.

## 4. 앱 완성도 — 출시 가능한가?

**현재 평가: 출시 불가. 내부 사용 또는 클로즈드 베타 수준.**

이유:
- 가격 오염 방어 로직이 오늘 막 붙었고 실운영 검증 전.
- Flutter 앱 UI/UX 재설계가 TODO로 명시되어 있음. 현재 UI는 기능 구현 중심의 프로토타입 수준.
- 거래 기능(찜, 판매 상태 변경, 구매/판매 이력, 판매자 프로필)이 미완.
- FCM 푸시 알림, 통합 검색 미구현.
- 자동화 테스트 없음 — 새 기능 추가 시마다 기존 기능이 깨질 위험.

반면, **개인 시세 조회 + 자산 관리** 용도로는 오늘 기준으로도 쓸 수 있는 수준. 카드 가격 조회 + 포트폴리오 파악이라는 핵심 기능은 작동함.

## 5. 다음 가장 중요한 3가지 개선 포인트

### 1위: eBay API 교차검증 실운영 검증

**이유**: 오늘 구현한 가격 오염 방지 guard의 핵심이 eBay API 호출임. 코드만 있고 실제로 API가 정상 응답하는지 검증이 안 된 상태. 방어막이 있는 척하지만 실제로 뚫려 있을 수 있음. 다음 세션 첫 번째 작업으로 권장.

### 2위: 437개 미매핑 카드 scrydex 매핑

**이유**: 매핑이 안 된 카드는 KO 예상가가 없어서 시세 화면에서 사실상 죽은 카드임. 437개 중 고가 희귀 카드가 포함될수록 앱의 핵심 가치(시세 조회)가 훼손됨. scrydex_mapper.html 작업이 이미 준비되어 있으니 집중 처리로 해결 가능.

### 3위: 번들 매물 필터링 로직 구현

**이유**: KO 예상가 계수 계산의 신뢰도가 이 데이터에 달려 있음. 번들 가격이 단품 가격으로 잡히면 계수가 낮게 계산되고 전체 KO 예상가가 왜곡됨. 데이터 품질의 구조적 문제.
