# 호가창 리디자인 — 레퍼런스 & UX 설계

> 작성: 2026-05-18 (Claude + 사용자 의도 정리)
> 대상: 카드 상세 호가창 + 매수/매도 흐름

## 1. 핵심 컨셉 (사용자 명시)

```
주식/코인 거래소 호가창 스타일
+
호가 셀 클릭 → 그 가격에 등록된 매수자/매도자 리스트 노출
```

= **StockX 모델**.
일반 주식/코인 호가창은 잔량만 표시하지만, P2P 카드 거래는 누가 걸어놨는지 알아야 채팅/거래 가능 → 호가 셀이 "그룹"이고 클릭 시 "구성원 리스트"가 펼쳐지는 hybrid.

## 2. 레퍼런스 (1~3순위)

### 1순위 — Hybrid 매칭형 (PokeFolio가 가야 할 방향과 거의 동일)

| 앱 | URL | 핵심 패턴 |
|---|---|---|
| **KREAM** | kream.co.kr (앱) | 한국 한정판 거래소. **사용자가 이미 메타몽 프로모 시세 수집에 KREAM 활용 중**. 카드 상세 → 매수/매도 호가 분리 표시, 호가별 가격 + 등록 수량, 즉시 거래/호가 등록 분기, 채팅 X (자체 결제) |
| **StockX** | stockx.com | 글로벌 한정판. **Bids / Asks 양방향 호가**, 가장 높은 매수("Highest Bid") + 가장 낮은 매도("Lowest Ask") 강조, 호가 클릭 → 즉시 거래 또는 새 호가 등록 |
| **GOAT** | goat.com | 운동화 중심. 호가 + 상태별 분리 (DS / Used) — 카드의 RAW / PSA10 등급 분리와 동일 구조 |
| **Soldout** | soldout.co.kr | KREAM 경쟁사, 모바일 UX 차이 비교용 |

→ **PokeFolio = KREAM (한국 친숙도) + StockX (양방향 호가)**. 단 거래는 **채팅으로 협상** (P2P 카드 시장 특성, 결제 시스템 없음).

### 2순위 — 모바일 호가 UI 디자인 (시각 패턴)

| 앱 | 핵심 |
|---|---|
| **토스증권** | 한국 모바일 호가 UI 표준. 깔끔한 폰트, 매수/매도 색 분리, 잔량 막대 그래프 |
| **업비트** | 코인 호가 모바일 UX. 가격 row 컴팩트, 손가락 친화 |
| **빗썸** | 업비트와 비교 — 더 컴팩트, 정보 밀도 높음 |
| **카카오페이증권** | 토스 다음 모바일 친화 디자인 |

→ **색상 규약 (한국 표준)**: 매수 = 빨강 / 매도 = 파랑. (한국 증시 관습 — 미국/글로벌은 반대. PokeFolio 타겟이 한국 사용자이므로 한국 관습.)

### 3순위 — TCG 도메인 거래 (카드 특화 UX)

| 앱 | 핵심 |
|---|---|
| **TCGplayer Marketplace** | 카드별 매도자 리스트. 각 매도자 행: 상태(NM/LP/MP) + 가격 + 판매자 평점 + 발송기간 |
| **Cardmarket** (EU) | 유럽 TCG. 가격 정렬, 상태 필터, 판매자 국가/언어 표시 |
| **Channel Fireball Marketplace** | 미국 TCG. 단순한 리스트, 카트 시스템 |

→ **카드 거래 row 정보**: 가격 / 상태(RAW·PSA·BRG + 등급) / 판매자 평점 / 사진 / 등록일. 채팅 버튼 제일 오른쪽. (CGC/BGS 미지원)

## 3. PokeFolio 적용 UX (ASCII mockup)

### 카드 상세 화면 — 호가창 영역

```
┌──────────────────────────────────────┐
│  [카드 이미지 + 카드명 + 시세 칩]      │
├──────────────────────────────────────┤
│  KO 추정가  ₩ 152,000   ▲ +2.4%       │ ← 대표 시세 (Sticky 헤더)
│  매도 12건  ·  매수 5건                │
├──────────────────────────────────────┤
│  [호가창]                              │
│                                       │
│   180,000  ▎▎▎▎ 3건  [매도]            │ ← 가장 비싼 매도
│   175,000  ▎▎ 2건    [매도]            │
│   170,000  ▎▎▎▎▎▎ 7건 [매도]           │ ← Lowest Ask (강조)
│  ────────────────────────────────     │
│         152,000  ← 현재가 (KO 추정)    │
│  ────────────────────────────────     │
│   140,000  ▎▎▎ 3건  [매수]             │ ← Highest Bid (강조)
│   130,000  ▎ 1건    [매수]             │
│   120,000  ▎ 1건    [매수]             │
│                                       │
├──────────────────────────────────────┤
│  [내가 팔기]      [내가 사기]          │ ← 호가 등록 (모달)
└──────────────────────────────────────┘
```

**디자인 규약**:
- 매도 = 파란색 row (위쪽, 가격 내림차순)
- 매수 = 빨간색 row (아래쪽, 가격 내림차순)
- 막대 길이 = 등록 건수 (해당 가격대 잔량 시각화)
- Lowest Ask + Highest Bid 강조 (배경 진하게 / 굵게)
- 현재가 = KO 추정가 (시세 정책)

### 호가 셀 클릭 시 — 등록 리스트 모달

```
┌──────────────────────────────────────┐
│  170,000원 매도 호가 (7건)            │  ← 클릭한 행 정보
│                                       │
│  ┌──────────────────────────────┐    │
│  │ 판매자A · ⭐4.8 · RAW         │    │
│  │ "민트급, 슬리브+토퍼"          │    │
│  │ 등록 5분 전                    │    │
│  │              [💬 채팅하기]    │    │
│  ├──────────────────────────────┤    │
│  │ 판매자B · ⭐4.5 · PSA 9       │    │
│  │ "감정카드, 슬랩 사진 있음"     │    │
│  │ 등록 1일 전                    │    │
│  │              [💬 채팅하기]    │    │
│  ├──────────────────────────────┤    │
│  │ 판매자C · — · RAW             │    │
│  │ (메모 없음)                    │    │
│  │ 등록 3일 전                    │    │
│  │              [💬 채팅하기]    │    │
│  │ ...                            │    │
│  └──────────────────────────────┘    │
│                                       │
│  [같은 가격에 매도 등록]              │
└──────────────────────────────────────┘
```

**리스트 row 정보**:
- 판매자 닉네임 + 평점 (없으면 "—")
- 카드 상태 (RAW / PSA10 등급 / BRG 등급) — CGC/BGS 미지원
- 메모 (사진 슬리브 상태 등)
- 등록 시각 (상대시간)
- **채팅하기 버튼** — 클릭 → 자동 채팅방 생성 (BuyOrder.userId ↔ Asset.userId 매칭, ROADMAP.md 명시)

## 4. 결정 사항 (2026-05-18 새벽 사용자 확정)

### 4-1. 동적 tick 단위 (1,000원 고정 ❌)

이유: 캡처에 4,375만원 카드 있음. 1,000원 단위로 쪼개면 의미 없음. 가격대별 자동 변경.

| 대표 시세 구간 | 호가 단위 |
|---:|---:|
| < 100,000원 | **1,000원** |
| 100,000 ~ 1,000,000원 | **5,000원** |
| 1,000,000 ~ 10,000,000원 | **10,000원** |
| 10,000,000원 이상 | **100,000원** |

→ 가격 입력 모달에서도 동일 tick 강제 (그룹 규약과 일치).

### 4-2. RAW / 감정등급 분리 (필수)

이유: RAW와 PSA10은 5~50배 차이. 섞이면 시세 왜곡 + 거래 사고. (메모리 `feedback_grading_companies.md`: PSA/BRG만. CGC/BGS 금지.)

호가창 상단 필터 chip:
```
[RAW] [PSA 10] [BRG]
```

1차 출시 단순화:
- **기본값 = RAW** (호가창 RAW 호가만)
- 감정카드는 chip 클릭 시 별도 호가창

### 4-3. 표시 행 / 리스트 / 거래 / 색상

| 항목 | 결정 |
|---|---|
| 호가창 표시 행 수 | 매도 5 + 매수 5 |
| 리스트 등장 방식 | 하단 시트 (Bottom sheet) |
| 즉시 거래 | 1차 채팅만, 결제 X |
| 색상 — 매도 | 파랑 (cool, 차분) |
| **색상 — 매수** | **초록** (한국 증시 빨강 대신, "긍정적 행동" 직관성) |
| 가격 입력 단위 | 동적 tick과 동일 자동 적용 |
| 상태 구분 | RAW 기본 + 감정등급 chip 필터 |

→ 빨강은 카드 거래에서 "경고/오류"로 해석될 수 있어 회피.

### 4-4. 거래 탭 화면 우선순위 재정렬

기존 (지금 캡처): "이 카드 판매 중" 박스가 메인 → 비어보임.

**새 구조**:
```
대표 시세 카드
  ↓
거래 요약 (매도 최저 / 매수 최고 / 스프레드 / 등록 N건)
  ↓
호가창 (매도 5 / 현재가 / 매수 5)
  ↓
[판매 호가 등록] [매수 호가 등록]
  ↓
(판매 리스트는 호가 row 클릭 시 하단 시트로만 노출)
```

호가 0건일 때 **빈 상태 CTA**:
```
아직 등록된 호가가 없습니다.
첫 번째 판매자/매수자가 되어보세요.
[판매 호가 등록] [매수 호가 등록]
```

## 5. 작업 Split (수면 후 진행 순서)

### Phase A — 디자인 확정 (15분)
- 색상 hex 확정: 매도 파랑(`#1E88E5` 또는 앱 톤 매치) / 매수 초록(`#10B981` 또는 앱 톤)
- 동적 tick 함수 spec (`hogaTick(price) → tickUnit`)
- 빈 상태 / 호가 0건 CTA 문구 확정
- Codex와 함께 mockup refine (CLAUDE.md 원칙)

### Phase B — 데이터 모델 보완 (30분)
- `BuyOrder` / `TradePost`에 `card_status` 필드 (RAW / PSA10 등) 확인. 없으면 추가.
- API: `GET /api/cards/{cardId}/hoga?status=RAW&top=5` (status 필터 + top N 행)
- 그룹 응답 DTO: `[{price: 170000, side: ask, count: 7}, ..., {price: 140000, side: bid, count: 3}, ...]`
- 동적 tick으로 price grouping은 백엔드에서 처리 (clientside 일관성 위해)
- 호가 row 클릭 시: `GET /api/cards/{cardId}/hoga/{price}?status=RAW&side=ask` (row 상세)

### Phase C — 호가창 위젯 (Flutter, 2~3시간)
- `card_detail_screen.dart`의 기존 호가 영역 위젯화: `HogaBoard`
- 매도/매수 row + 막대 그래프
- Lowest Ask / Highest Bid 강조
- 현재가 sticky 위치

### Phase D — 리스트 시트 (1~2시간)
- 호가 셀 클릭 → `showModalBottomSheet`
- API: `GET /api/cards/{cardId}/hoga/{price}?side=ask|bid` 
- row 위젯 (판매자 + 상태 + 평점 + 메모 + 채팅 버튼)

### Phase E — 채팅 자동 생성 (1시간)
- "채팅하기" 클릭 → 채팅방 lookup or 생성
- BuyOrder.userId ↔ Asset.userId 매칭
- 첫 메시지 자동 (카드명/가격/상태)

### Phase F — 호가 등록 모달 (1시간)
- "내가 팔기" / "내가 사기" 버튼 → 모달
- 가격 입력 (1,000원 단위) + 상태 + 메모 + (선택) 사진

### Phase G — 검증 + Codex 리뷰 (30분, CLAUDE.md 원칙)

총 예상: **6~8시간** (수면 후 하루치)
