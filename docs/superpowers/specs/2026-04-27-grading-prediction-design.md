# 카드 등급 예측 기능 설계 (2026-04-27)

## 개요

포켓몬 카드 사진 10장을 입력받아 PSA/CGC 스타일의 예상 등급을 소수점 1자리 수치로 반환하는 기능.
하단 탭 "채팅" 자리를 "등급"으로 교체. 외부 Vision API 없이 Python OpenCV만 사용.

---

## 아키텍처

```
Flutter → Spring Boot (8080) → Python FastAPI (8081)
```

- Flutter: 10장 단계별 촬영 UI + 결과 표시
- Spring Boot: 사진 수신, FastAPI 프록시, 결과 저장
- Python FastAPI: OpenCV 기반 이미지 분석 엔진

Spring Boot ↔ FastAPI는 내부 통신(localhost). 사용자는 Spring Boot 엔드포인트만 사용.

---

## 입력: 10장 사진

| 순서 | 사진 | 분석 목적 |
|------|------|-----------|
| 1 | 앞면 전체 | 센터링 |
| 2 | 뒷면 전체 | 화이트닝 |
| 3~6 | 앞 모서리 4개 (좌상/우상/좌하/우하) | 코너 마모 |
| 7~10 | 뒤 모서리 4개 (좌상/우상/좌하/우하) | 코너 마모 + 화이트닝 |

Flutter UI에서 각 단계마다 어떤 각도/거리로 찍어야 하는지 오버레이 가이드 표시.

---

## 분석 로직 (Python OpenCV)

### 1. 센터링 (앞면 전체)
- 카드 외곽선 감지 (Canny + Hough Lines)
- 내부 인쇄 영역 경계 감지
- 상하/좌우 여백 비율 계산
- 기준: 55/45 이하 → 감점, 50/50에 가까울수록 만점

### 2. 코너 마모 (모서리 8장)
- Harris Corner Detection으로 코너 선명도 측정
- 코너 라운딩(마모) 정도 수치화
- 8장 평균 점수 산출

### 3. 표면 스크래치 (앞/뒤 전체)
- Canny 엣지 감지 후 카드 내부 선형 노이즈 추출
- Hough Line Transform으로 스크래치성 직선 패턴 감지
- 노이즈 밀도로 점수 산출

### 4. 화이트닝 (뒷면 전체 + 뒤 모서리 4장)
- HSV 변환 후 엣지/코너 영역 흰색/회색 픽셀 비율 측정
- 포켓몬 카드 뒷면 기준 파란색(HSV ~210°) 대비 편차 감지
- 화이트닝 면적 비율로 점수 산출

---

## 점수 산출

| 항목 | 가중치 | 분석 소스 |
|------|--------|-----------|
| 센터링 | 15% | 앞면 1장 |
| 코너 | 35% | 모서리 8장 |
| 표면 | 25% | 앞뒤 전체 2장 |
| 화이트닝 | 25% | 뒷면 + 뒤 모서리 |

```
종합 점수 = 각 항목 점수(1~10) × 가중치 합산
→ 소수점 1자리 반올림 (예: 9.2)
```

화이트닝이 심한 경우(점수 4 이하) 종합 점수에 추가 페널티 적용.

## 결과 표시 (Flutter)

```
┌──────────────────────────┐
│  예상 등급               │
│       9.1 / 10.0         │
├──────────────────────────┤
│  센터링    9.5  ●●●●●   │
│  코너      8.8  ●●●●○   │
│  표면      9.0  ●●●●●   │
│  화이트닝  9.2  ●●●●●   │
└──────────────────────────┘
  PSA 9 / CGC 9.5 수준으로 예상됩니다
```

항목별 점수 + 시각적 바 + 종합 점수 + 참고용 PSA/CGC 환산 멘트.

## 데이터 저장

신규 테이블 `grading_results`:

```sql
CREATE TABLE grading_results (
  result_id       VARCHAR(50) PRIMARY KEY,
  user_id         VARCHAR(50) NOT NULL,
  card_id         VARCHAR(50),
  centering_score NUMERIC(3,1),
  corner_score    NUMERIC(3,1),
  surface_score   NUMERIC(3,1),
  whitening_score NUMERIC(3,1),
  total_score     NUMERIC(3,1),
  created_at      TIMESTAMP NOT NULL DEFAULT now()
);
```

사진 파일은 서버 저장 안 함 — 분석 후 즉시 폐기. 점수만 저장.

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/grading/analyze` | 사진 10장 업로드 → 분석 → 결과 반환 |

## Flutter UI 흐름

```
[등급] 탭
  └─ 안내 화면 (기능 설명 + "시작하기")
       └─ 카드 선택 (선택사항, 나중에 결과와 연결용)
            └─ 단계별 촬영 (1/10 ~ 10/10, 각 단계 가이드 오버레이)
                 └─ 분석 중 로딩
                      └─ 결과 화면 (항목별 + 종합)
                           └─ 저장 / 다시 분석
```
