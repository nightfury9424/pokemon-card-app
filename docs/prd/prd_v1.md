# 포켓몬 카드 앱 PRD v1

## 1. 문서 목적
이 문서는 포켓몬 카드 앱의 제품 방향, 핵심 기능, MVP 범위, 사용자 흐름, 도메인 구조를 다시 정리한 제품 기획 문서다.

이 문서는 아래 제품 정체성을 기준으로 작성한다.

**포켓몬 카드를 스캔해 정확한 카드를 식별하고, 시세를 확인하고, 보유 자산으로 등록·관리할 수 있는 카드 자산 앱**

또한 기능 우선순위는 아래처럼 나눈다.

- **1차 목표**: 스캔, 카드 식별, 카드 정보, 시세 확인, 자산 등록 및 포트폴리오
- **2차 목표**: 판매 공개, 다른 사용자 포트폴리오 조회, 사용자 간 채팅방 생성 및 거래 연결

---

## 2. 서비스 한 줄 정의

### 제품 정의
**포켓몬 카드를 주식처럼 조회하고 관리하는 스캔 기반 시세·자산 앱**

### 확장 정의
**카드를 스캔하면 종목처럼 식별되고, 시세를 확인할 수 있으며, 내 자산으로 등록해 포트폴리오를 관리할 수 있는 서비스**

### 레퍼런스 포지셔닝
**Monprice 앱의 한국판** — Monprice가 영어권 기반의 포켓몬 카드 스캔·시세 앱이라면, 본 서비스는 한국 거래 채널(당근, 번개장터, icu.gg) 기반의 시세 데이터와 한국 사용자 경험에 최적화된 동일 개념의 앱이다.

---

## 3. 제품의 본질

이 앱은 단순히 카드 정보를 검색하는 앱이 아니다.

이 앱의 핵심은 아래 4가지다.

1. **스캔 중심 카드 식별**
2. **시세 기반 가격 확인**
3. **포켓몬 카드를 자산처럼 등록·관리**
4. **향후 거래 확장이 가능한 구조 확보**

즉, 이 앱에서 카드는 단순 수집품이 아니라 **시세가 있고 관리 가능한 자산**이다.

## 4. 해결하려는 문제

### 4-1. 현재 사용자 불편
한국 포켓몬 카드 사용자는 아래 문제를 겪는다.

1. 내 카드가 정확히 어떤 카드인지 바로 확인하기 어렵다.
2. 카드가 어느 세트, 어느 팩에서 나온 카드인지 찾기 번거롭다.
3. 최근 얼마에 거래되는지 한눈에 보기 어렵다.
4. 보유 카드 수량, 매입가, 현재 가치, 손익을 체계적으로 관리하기 어렵다.
5. 국내 기준으로 카드 시세와 자산을 함께 다루는 앱이 부족하다.

### 4-2. 제품이 제공할 해결책
본 서비스는 아래 흐름을 제공한다.

**카드 스캔 → 카드 식별 → 세트/팩 정보 확인 → 최근 시세 확인 → 내 자산 등록 → 포트폴리오 확인**

2차 확장에서는 아래 흐름을 추가한다.

**판매 의사 설정 → 공개 카드 노출 → 다른 사용자 포트폴리오 확인 → 채팅방 생성 → 거래 연결**

## 6. 제품 목표

### 6-1. 1차 목표
- 카드를 스캔만으로 식별할 수 있어야 한다.
- 카드가 어느 세트/팩 소속인지 바로 확인할 수 있어야 한다.
- 최근 시세를 확인할 수 있어야 한다.
- 보유 카드를 자산처럼 등록할 수 있어야 한다.
- 내 포트폴리오에서 총 자산, 손익, 카드별 보유 현황을 볼 수 있어야 한다.

### 6-2. 2차 목표
- 판매 의사가 있는 카드를 공개 상태로 바꿀 수 있어야 한다.
- 다른 사용자의 공개 포트폴리오 또는 공개 카드 목록을 볼 수 있어야 한다.
- 거래 의사가 생기면 앱 내 채팅방을 생성할 수 있어야 한다.

### 6-3. 개발 관점 목표
- 1차 MVP에서 스캔 결과 기반 카드 상세, 시세, 자산 등록, 포트폴리오까지 연결
- 2차에서 공개 카드/공개 포트폴리오/채팅 기반 거래 연결 추가

## 8-3-1. 시세 데이터 수집 원칙

본 서비스의 시세 데이터는 당근, 번개장터, icu.gg 등 국내 거래 채널의 데이터를 수집·정규화하여 산출한다.

실시간 직조회만으로 처리하지 않고, 아래 파이프라인 구조를 기본으로 한다.

**원천 데이터 수집 → 카드 식별/매핑 → 노이즈 제거 → 시세 계산 → 그래프 제공**

### 수집 채널
- 당근마켓
- 번개장터
- icu.gg

### 파이프라인 단계
1. **수집**: 각 채널에서 거래 데이터 크롤링/수집 (Python 담당)
2. **카드 매핑**: 수집된 게시글 제목·이미지를 카드 마스터 DB와 매핑
3. **노이즈 제거**: 이상 거래가, 중복 데이터, 미매핑 데이터 필터링
4. **시세 계산**: 중앙값·평균가·최저/최고가 산출
5. **그래프 제공**: 기간별 시세 추이 데이터 저장 및 API 제공

## 8-4. 자산 등록
사용자가 식별된 카드를 내 자산으로 등록하는 기능

### 입력 항목
- 보유 수량
- 매입가
- 메모
- 등록일

### 카드 상태 처리 방식
카드 상태는 사용자가 직접 세부 입력하는 방식이 아니라, **스캔 결과를 기반으로 자동 판별**하는 방식을 우선 적용한다.

본 서비스는 카드 상태를 크게 아래 2가지로 구분한다.

- **일반 카드 (RAW)**: 공식 그레이딩을 받지 않은 일반 카드
- **그레이딩 카드 (GRADED)**: PSA, BGS, CGC 등 공식 감정을 받은 카드

카드를 스캔했을 때 카드 외형과 그레이딩 케이스/라벨 정보를 기준으로 일반 카드인지, 그레이딩 카드인지 우선 판별한다.

그레이딩 카드로 판별된 경우에는 아래 정보를 함께 인식하거나 표시 대상으로 삼는다.

- 그레이딩 기관
- 등급 점수
- 인증번호

즉, 사용자가 상태를 세부적으로 직접 기입하는 것이 아니라,
**스캔 결과를 통해 일반 카드인지 그레이딩 카드인지 자동 구분하고, 그레이딩 카드면 기관과 점수를 함께 표시하는 구조**를 기본 정책으로 한다.

### 역할
- 카드 시세 확인 이후 바로 자산 등록으로 이어짐
- 단순 조회 앱이 아니라 자산관리 앱으로 전환되는 핵심 기능
- 상태 정보 입력 부담을 줄이고 스캔 기반 UX를 강화함

### 우선순위
최상

## 10. 1차 MVP 정의

### 10-1. 1차 MVP 목표
사용자가 **카드를 스캔하고, 정확한 카드 정보를 확인하고, 최근 시세를 보고, 자산으로 등록하고, 포트폴리오를 관리할 수 있는 상태**를 만든다.

### 10-2. 1차 MVP 포함 범위
- 카드 마스터 DB 구축
- 스캔 결과 기반 카드 식별 구조
- 카드 상세 API
- 시세 조회 API
- 보유 자산 등록/수정/삭제 API
- 포트폴리오 API
- Flutter 화면 구현
- 검색 fallback 기능

### 10-3. 1차 MVP 제외 범위
- 공개 카드 노출
- 공개 포트폴리오
- 앱 내 채팅
- 거래 완료 처리
- 대규모 다중 사용자 거래 기능

---

## 11. 2차 확장 목표

### 11-1. 2차 목표
사용자가 **내 자산 중 공개 가능한 카드를 노출하고, 다른 사용자의 공개 카드 또는 공개 포트폴리오를 보고, 거래 의사가 생기면 채팅방을 만들어 연결할 수 있는 상태**를 만든다.

## 12. 사용자 핵심 플로우

## 12-1. 1차 스캔 자산 플로우
1. 사용자가 카드를 스캔한다.

### Asset 도메인 상태 설계 원칙
Asset의 카드 상태는 사용자가 NM, LP 같은 세부 상태를 직접 입력하는 구조가 아니라, **스캔 결과를 기반으로 자동 판별하는 구조**를 우선 적용한다.

#### 상태 분류
- `RAW` : 일반 카드
- `GRADED` : PSA/BGS/CGC 등 그레이딩 카드

#### 그레이딩 카드 추가 정보
- `gradingCompany`
- `gradeValue`
- `certNumber`

#### 설계 원칙
- 사용자는 자산 등록 시 상태를 직접 세부 기입하지 않는다.
- 스캔 결과를 통해 일반 카드/그레이딩 카드 여부를 우선 판별한다.
- 그레이딩 카드면 기관과 점수를 함께 저장·표시한다.
- 초기 버전에서는 자동 판별 정확도 한계를 고려해 필요 시 보정 선택 기능을 둘 수 있다.

## 20. 최종 정리

이 앱의 1차 본질은 아래와 같다.

**포켓몬 카드를 스캔해 카드 정보를 식별하고, 세트/팩과 시세를 확인하고, 자산으로 등록해 포트폴리오를 관리하는 앱**

여기서 자산 등록 시 카드 상태는 사용자가 일일이 세부 입력하는 것이 아니라,
**스캔 결과를 통해 일반 카드와 그레이딩 카드를 자동 구분하고, 그레이딩 카드면 PSA/BGS 등의 기관과 등급을 함께 표시하는 방식**을 기본으로 한다.

이 앱의 2차 확장 방향은 아래와 같다.

**판매 의사가 있는 카드와 공개 포트폴리오를 다른 사용자에게 노출하고, 거래를 원하면 채팅방으로 연결하는 카드 자산 플랫폼**
