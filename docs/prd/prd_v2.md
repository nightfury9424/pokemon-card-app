# 포켓몬 카드 앱 PRD v2

> v1 대비 변경 사항: 시세 다중소스 구조, 포트폴리오 공개 정책, 내부 거래 플로우, 스캔 기술 구체화, 푸시 알림, 중복 스캔 처리, DB 스키마 보완

---

## 1. 문서 목적

이 문서는 포켓몬 카드 앱의 제품 방향, 핵심 기능, MVP 범위, 사용자 흐름, 도메인 구조를 정리한 제품 기획 문서다.

제품 정체성:

**포켓몬 카드를 스캔해 정확한 카드를 식별하고, 시세를 확인하고, 보유 자산으로 등록·관리하며, 개인 간 직거래까지 연결할 수 있는 카드 자산 플랫폼**

기능 우선순위:

- **1차 목표**: 스캔, 카드 식별, 카드 정보, 시세 확인, 자산 등록, 포트폴리오 관리
- **2차 목표**: 포트폴리오 공개, 다른 사용자 포트폴리오 조회, 판매 카드 탐색, 채팅방 기반 개인 간 거래 연결

## 2. 서비스 정의

### 제품 정의
**포켓몬 카드를 주식처럼 조회하고 관리하는 스캔 기반 시세·자산·거래 플랫폼**

### 확장 정의
**카드를 스캔하면 종목처럼 식별되고, 다양한 채널의 시세를 확인할 수 있으며, 내 자산으로 등록해 포트폴리오를 관리하고, 팔고 싶은 카드는 다른 사용자와 직접 연결해 거래할 수 있는 서비스**

### 레퍼런스 포지셔닝
**Monprice 앱의 한국판**
Monprice가 영어권 기반의 포켓몬 카드 스캔·시세 앱이라면, 본 서비스는 한국 거래 채널 기반의 시세 데이터와 한국 사용자 경험에 최적화된 동일 개념의 앱이다.

## 3. 제품의 본질

이 앱의 핵심은 아래 4가지다.

1. **스캔 중심 카드 식별** — 카메라로 카드를 찍으면 바로 어떤 카드인지 안다
2. **시세 기반 가격 확인** — 국내 다양한 채널에서 수집한 현재 시세를 바로 본다
3. **포켓몬 카드를 자산처럼 등록·관리** — 수량, 매입가, 평가가, 손익을 관리한다
4. **개인 간 거래 확장 구조** — 자산에서 판매로, 카드 중심의 C2C 거래 플랫폼으로 확장

카드는 단순 수집품이 아니라 **시세가 있고 관리 가능하며 거래 가능한 자산**이다.

## 4. 해결하려는 문제

### 4-1. 현재 사용자 불편

1. 내 카드가 정확히 어떤 카드인지 바로 확인하기 어렵다.
2. 카드가 어느 세트, 어느 팩에서 나온 카드인지 찾기 번거롭다.
3. 현재 얼마에 거래되는지 한눈에 보기 어렵다.
4. 보유 카드 수량, 매입가, 현재 가치, 손익을 체계적으로 관리하기 어렵다.
5. 국내 기준으로 카드 시세와 자산을 함께 다루는 앱이 부족하다.
6. 카드를 팔고 싶어도 신뢰할 수 있는 거래 채널이 분산되어 있다.

### 4-2. 제품이 제공할 해결책

**1차 플로우**
카드 스캔 → 카드 식별 → 세트/팩 정보 확인 → 현재 시세 확인 → 내 자산 등록 → 포트폴리오 확인

**2차 플로우**
판매 의사 설정 → 포트폴리오 공개 → 다른 사용자 포트폴리오 탐색 → 채팅방 생성 → 거래 연결 → 거래 완료 확인 → 내부 거래가 기록

## 6. 제품 목표

### 6-1. 1차 목표
- 카드를 스캔만으로 식별할 수 있어야 한다.
- 카드가 어느 세트/팩 소속인지 바로 확인할 수 있어야 한다.
- 다양한 국내 채널 기반의 현재 시세를 확인할 수 있어야 한다.
- 보유 카드를 자산으로 등록할 수 있어야 한다.
- 포트폴리오에서 총 자산, 손익, 카드별 보유 현황을 볼 수 있어야 한다.

### 6-2. 2차 목표
- 포트폴리오를 공개 전환할 수 있어야 한다.
- 카드 단위로 판매 의사를 설정할 수 있어야 한다.
- 다른 사용자의 공개 포트폴리오와 판매 카드를 볼 수 있어야 한다.
- 거래 의사가 생기면 앱 내 채팅방을 만들 수 있어야 한다.
- 거래 완료 시 양방향 확인으로 거래가가 기록되어야 한다.

## 7. 제품 핵심 가치

### 7-1. 스캔 중심성
사용자는 검색보다 스캔으로 진입하는 경험을 더 자주 사용한다.

### 7-2. 카드의 자산화
카드를 단순 보관하는 것이 아니라, 수량·매입가·평가가·손익까지 관리한다.

### 7-3. 시세 확인의 편의성
국내 다양한 채널(네이버 쇼핑, 번개장터, 네이버 카페, icu.gg 등)의 데이터를 통합해 카드별 현재 시세를 빠르게 확인한다.

### 7-4. 개인 간 거래 플랫폼
초기에는 자산관리 중심으로 시작하되, 공개 포트폴리오와 채팅 기반 C2C 거래로 확장한다.

## 8. 핵심 기능 정의

### 8-1. 카드 스캔 및 식별

사용자가 카메라를 카드에 갖다 대면 실시간으로 카드를 인식하는 기능

#### 기술 파이프라인
1. **카드 감지** (detector): 프레임에서 카드 외곽선 감지 → 원근 보정(CamScanner 방식) → 630x880 정규화
2. **OCR** (ocr): 상단 15%(카드명) + 하단 15%(컬렉션 번호) EasyOCR 추출
3. **DB 퍼지 매칭** (matcher): 전체 카드 메모리 로드 후 RapidFuzz 점수 계산 → Top 3 후보 선별
   - 컬렉션 번호 매칭: 최대 60점
   - 카드명 퍼지 매칭: 최대 40점
4. **이미지 유사도 검증** (image_compare): SIFT 특징점 기반 후보 이미지 대조 → 최종 확정
5. **결과 표시**: 카드명 + 번호 오버레이, 등록 버튼 표시

#### 처리 구조
- 카메라 60fps는 메인 스레드, AI 연산은 백그라운드 스레드 (비동기)
- 초점 감지(Laplacian variance) 후 스캔 트리거
- 이미 보유 중인 카드 스캔 시: "이미 보유 중인 카드예요. 추가 등록할까요?" 안내

#### 주요 요구사항
- 스캔 성공 → 바로 카드 상세로 이동, 등록 버튼 제공
- 자동 인식 보완용 후보 리스트 제안
- 스캔 실패 시 검색 fallback 안내

#### 우선순위: 최상

---

### 8-2. 카드 상세 정보 조회

#### 표시 정보
- 카드명, 카드번호, 레어도
- 카드 이미지 (S3/CDN 저장본)
- 세트명, 팩명, 발매 정보
- 언어 (KO / JA / EN)
- 일러스트레이터

#### 화면 구성 (스캔 결과 화면)
- 카드 이미지 + 기본 정보
- 현재 시세 한 줄 요약 (외부 시세)
- 등록 버튼

#### 화면 구성 (카드 상세 화면)
- 카드 전체 정보
- 시세 그래프 (외부 시세, 추후 내부 거래가 추가)
- 자산 등록 / 판매 의사 설정

#### 우선순위: 최상

---

### 8-3. 시세 조회

#### 시세 데이터 구조

**1차: 외부 시세 (다중 소스 집계)**
국내 거래 채널에서 수집한 데이터를 정규화해 현재 시세를 산출한다.

| 소스 | 방식 | 신뢰도 | 비고 |
|------|------|--------|------|
| 네이버 쇼핑 API | 공식 API (무료) | 높음 | 현재 판매가 |
| 크림(KREAM) | 크롤링 | 매우 높음 | 실거래가(체결가) 기록, KO 카드 포함 |
| 번개장터 | 크롤링 | 중간~높음 | 현재 판매가 |
| 네이버 카페 검색 API + 본문 파싱 | 공식 API + 크롤링 | 중간 | 실거래 텍스트 파싱 |
| icu.gg | 크롤링 | 중간 | 글로벌 집계가 |
| 중고나라 | 크롤링 | 중간 | 현재 판매가 |

- 소스별 신뢰도 가중치 적용 후 집계
- 크롤링 소스는 ToS 검토 후 운영
- "현재 시세" 표현 사용 (실거래가 아닌 현재 판매가 기반)

**2차: 내부 거래가 (앱 내 거래 완료 기록)**
앱 내에서 거래가 완료된 경우 양방향 확인으로 기록된 실거래가.
거래량이 쌓인 이후 별도 데이터 소스로 그래프에 추가.

#### 표시 정보
- 현재 시세 대표값
- 최근 7일 중앙값
- 최근 30일 평균가
- 최저가 / 최고가
- 데이터 건수
- 시세 추이 그래프

#### 시세 파이프라인
1. **수집**: 각 채널 크롤링/API 호출
2. **카드 매핑**: 수집 데이터를 카드 마스터 DB와 매핑
3. **노이즈 제거**: 이상가, 중복, 미매핑 데이터 필터링
4. **시세 계산**: 가중 중앙값·평균가·최저/최고가 산출
5. **집계 캐싱**: price_summaries 테이블에 7D/30D 집계 저장

#### 우선순위: 최상

---

### 8-4. 자산 등록

#### 입력 항목
- 보유 수량 (스캔 시 기본 1, 기존 보유 카드면 추가 여부 선택)
- 매입가
- 메모
- 등록일

#### 카드 상태
- **RAW**: 일반 카드 (스캔 기반 자동 판별)
- **GRADED**: 그레이딩 카드 (PSA/BRG, 케이스/라벨 인식)
  - 그레이딩 기관, 등급 점수, 인증번호 포함

#### 중복 스캔 처리
- 이미 자산으로 등록된 카드를 재스캔하면 "이미 보유 중인 카드예요" 안내
- 추가 등록 여부 선택 (같은 카드를 여러 매입가로 보유 가능)

#### 우선순위: 최상

---

### 8-5. 포트폴리오 / 자산 대시보드

#### 표시 정보
- 총 보유 카드 수
- 총 매입금액
- 총 평가금액 (현재 시세 기준)
- 전체 손익
- 카드별 손익
- 고가 카드 순위
- 자산 비중

#### 우선순위: 최상

---

### 8-6. 보조 검색 기능

카드명, 카드번호, 세트명으로 카드를 찾는 기능. 스캔 실패 시 fallback 진입점.

#### 우선순위: 중

---

### 8-7. 포트폴리오 공개 설정 (2차)

#### 정책
- **기본값: 비공개** — 내 포트폴리오는 기본적으로 나만 볼 수 있다
- 사용자가 포트폴리오 전체를 공개로 전환할 수 있다
- 공개 시 다른 사용자가 내 전체 보유 카드 목록을 볼 수 있다
- 포트폴리오 공개와 판매 의사 설정은 독립적으로 작동한다

#### 우선순위: 중 (2차)

---

### 8-8. 판매 의사 설정 (2차)

#### 정책
- 포트폴리오 공개 여부와 무관하게, 카드 단위로 "팔거야/안팔거야" 설정
- 판매 희망가는 외부 시세 기준으로 자동 제안, 사용자가 수정 가능
- 판매 의사를 설정한 카드는 다른 사용자의 포트폴리오 탐색에서 노출될 수 있음

#### 입력 항목
- 판매 의사 여부
- 판매 수량
- 희망 판매가 (외부 시세 자동 제안 → 사용자 수정 가능)
- 판매 메모

#### 우선순위: 중 (2차)

---

### 8-9. 공개 포트폴리오 및 판매 카드 탐색 (2차)

공개 설정된 다른 사용자의 포트폴리오를 탐색하고, 판매 의사 카드를 찾는 기능

#### 우선순위: 중 (2차)

---

### 8-10. 채팅방 생성 및 거래 연결 (2차)

거래 의사가 생긴 사용자끼리 앱 내 채팅방을 생성하는 기능

#### 거래 완료 플로우 (C안 - 양방향 확인)
1. 판매자 "거래완료 요청" + 최종 거래가 입력
2. 구매자 확인
3. 거래가 `price_snapshots`에 source=APP으로 기록
4. 거래량이 쌓이면 내부 시세 그래프 데이터로 활용

당근마켓처럼 채팅방까지 앱이 연결해주고, 실제 거래는 사용자 간 직접 진행. 결제 처리 없음.

#### 우선순위: 중 (2차)

---

### 8-11. 푸시 알림

#### 1차 알림
- 내 보유 카드 시세 X% 이상 변동 시 알림

#### 2차 알림
- 내가 찜한 카드의 판매자가 등장했을 때
- 채팅방 새 메시지
- 거래 완료 요청 수신

#### 우선순위: 중

## 9. 우선순위 정리

### 1차 핵심 기능

## 12. 사용자 핵심 플로우

### 12-1. 1차: 스캔 자산 플로우
1. 사용자가 카드를 스캔한다.
2. OCR + SIFT 분석으로 카드 후보를 식별한다.
3. 사용자가 정확한 카드를 확인(또는 선택)한다.
4. 카드 상세 정보와 현재 시세를 확인한다.
5. 자산 등록 버튼을 누른다. (이미 보유 중이면 추가 등록 여부 선택)
6. 포트폴리오에 반영된다.

### 12-2. 2차: 거래 연결 플로우
1. 사용자가 포트폴리오를 공개 전환한다.
2. 카드 단위로 "팔거야" 설정 + 희망가 입력 (외부 시세 자동 제안).
3. 다른 사용자가 공개 포트폴리오를 탐색한다.
4. 원하는 카드를 발견하고 채팅방을 생성한다.
5. 채팅으로 거래를 조율한다.
6. 판매자가 "거래완료 요청" + 최종 가격 입력 → 구매자 확인.
7. 거래가가 내부 시세 데이터로 기록된다.

## 22. 최종 정리

**1차 본질**
포켓몬 카드를 스캔해 카드 정보를 식별하고, 국내 다양한 채널의 현재 시세를 확인하고, 자산으로 등록해 포트폴리오를 관리하는 앱.

**2차 확장**
포트폴리오를 공개하고, 판매 의사 카드를 노출하고, 다른 사용자와 채팅방으로 연결해 직거래하는 카드 C2C 플랫폼. 내부 거래가가 쌓이면 자체 시세 데이터로 활용.

**핵심 리스크**
스캔 정확도가 앱의 첫 인상을 결정한다. 1단계에서 이미지 전체 수집 + pgvector 기반 유사도 검색 완성이 최우선이다.
